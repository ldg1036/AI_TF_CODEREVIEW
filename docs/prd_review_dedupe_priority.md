# 결과 중복 정리 및 우선순위 추천 PRD

- 작성일: 2026-03-06
- 상태: In Progress
- 담당: Codex

## 요약

현재 코드리뷰 프로그램은 P1/P2/P3 결과를 한 화면에 보여주지만, 사용자 입장에서는 동일 원인의 중복 이슈가 많고 무엇을 먼저 고쳐야 하는지 바로 판단하기 어렵다.  
이번 기능의 목표는 분석 결과를 더 적게, 더 명확하게, 더 실행 가능하게 보이도록 만드는 것이다.  
1차 범위에서는 기존 분석 엔진을 크게 바꾸지 않고 프런트 렌더링 단계에서 중복 효과를 요약하고, 우선 수정 대상 Top 5를 추천한다.

## 문제 정의

- 동일 파일/객체에서 유사한 이슈가 반복되어 사용자가 체감하는 결과량이 과도하다.
- P1/P2/P3가 함께 나와도 실제 수정 우선순위가 드러나지 않는다.
- 현재 대시보드는 전체 개수 중심이라 “어디부터 고칠지” 정보가 부족하다.

## 목표/비목표

- 목표:
  - 대시보드에서 중복 정리 효과를 보여준다.
  - 분석 직후 우선 수정 추천 Top 5를 보여준다.
  - 추천은 severity, source, duplicate count를 반영한 단순하고 설명 가능한 규칙으로 계산한다.
  - 기존 API 스키마를 바꾸지 않고 프런트 단에서 우선 구현한다.
- 비목표:
  - 백엔드 분석 규칙 자체를 재설계하지 않는다.
  - suppress/waive 같은 장기 운영 기능은 이번 범위에 포함하지 않는다.
  - 추천 결과를 바로 자동 수정으로 연결하지 않는다.

## 대상 사용자 및 사용 시나리오

- 대상 사용자:
  - 코드 리뷰 결과를 직접 수정하는 개발자
  - 릴리스 전 위험 구간만 빠르게 확인하려는 운영자/리더
- 사용 시나리오:
  - 사용자가 분석 버튼을 누른 뒤, 대시보드에서 “실제 손봐야 할 핵심 5개”를 먼저 본다.
  - 사용자는 결과 테이블 전체를 훑기 전에 hotspot 파일/객체를 파악한다.
  - 중복이 많이 접힌 경우, 전체 이슈 수와 실질 작업 대상 수의 차이를 이해한다.

## 성공 지표

- 분석 결과 직후 대시보드에 Top 5 추천이 항상 표시된다.
- 추천 항목은 최소 1개 이상, 최대 5개까지 표시된다.
- 중복 정리 요약에 `원시 이슈 수`, `표시 행 수`, `접힌 중복 수`가 표시된다.
- 기존 release gate, UI smoke, benchmark 흐름을 깨지 않는다.

## 범위(포함/제외)

- 포함:
  - 프런트 대시보드 카드 2종 추가
  - 결과 행 메타데이터 보강
  - 추천 점수 계산 로직 추가
  - 중복 접힘 요약 계산 추가
- 제외:
  - 저장형 baseline 비교
  - 오탐 suppression
  - 사용자 설정 가능한 점수 가중치

## 요구사항 (Must/Should/Could)

- Must:
  - 중복 정리 요약 카드 표시
  - 우선 수정 추천 Top 5 표시
  - 추천 기준은 severity, source, duplicate count를 반영
  - P1 review-truth 행 그룹핑과 충돌 없이 동작
- Should:
  - 추천 문구가 파일/객체 중심으로 이해 가능해야 함
  - 같은 객체의 유사 항목은 하나의 추천으로 묶음
- Could:
  - 추천 클릭 시 해당 결과로 이동
  - source별 색상/배지 차별화

## 동작 설계 (간단)

- 입력:
  - 기존 `analysisData.violations`
  - `buildWorkspaceRowIndex()`에서 생성되는 화면용 행
- 처리:
  - 각 행에 `duplicateCount`, `ruleIds`, `file`, `source`, `severity` 메타를 유지
  - 추천 그룹 키는 우선 `source + object/file`
  - 점수는 `severity weight + source weight + duplicate bonus`
  - 상위 5개 그룹을 추천 목록으로 표시
- 출력:
  - 대시보드 요약 카드
  - 추천 리스트

## 수용 기준 (AC)

- [ ] 분석 후 대시보드에 “중복 정리 요약” 섹션이 보인다.
- [ ] 분석 후 대시보드에 “우선 수정 추천 Top 5” 섹션이 보인다.
- [ ] 추천 목록은 5개를 넘지 않는다.
- [ ] 추천 항목은 객체/파일명, source, severity, 중복 개수 또는 이슈 수를 포함한다.
- [ ] 추천 카드에서 같은 hotspot / 같은 rule family 기준으로 결과를 바로 좁혀볼 수 있다.
- [ ] 기존 `node --check frontend/renderer.js`를 통과한다.
- [ ] 기존 UI smoke와 release gate 흐름을 깨지 않는다.

## 리스크 및 대응

- 리스크:
  - 프런트 추천 로직이 실제 중요도와 다르게 보일 수 있음
  - 기존 대시보드가 복잡해질 수 있음
- 대응:
  - 점수 규칙을 단순하게 유지하고 문구를 설명 가능하게 만든다.
  - 첫 단계에서는 읽기 전용 요약만 제공하고 자동 액션은 붙이지 않는다.

## 결정 사항

- 1차 구현은 API 변경 없이 프런트에서 계산한다.
- 추천 기준은 explainable heuristic으로 시작한다.
- 결과 테이블 자체의 대규모 구조 변경은 하지 않는다.
- 2차 구현에서는 severity/source/duplicate count 외에 객체 hotspot과 rule cluster 가중치를 포함한다.

## 미결 사항

- 추천 항목 클릭 시 해당 row로 스크롤 이동까지 연결할지
- 프로젝트별 가중치 커스터마이즈를 둘지
- P2/P3도 장기적으로 suppress/waive 대상에 포함할지
